# 20 — Transformer ブロック

## この章で作るもの

RMSNorm、因果的自己注意機構 (causal self-attention)、残差接続 (residual
connection)、位置ごとのフィードフォワードネットワークを組み合わせて、形状を
保存する pre-norm ブロックを構成します。ソース:
`src/main/scala/learnai/transformer/TransformerBlock.scala`。

## ブロックの式

$$
H=X+\operatorname{Attention}(\operatorname{RMSNorm}(X))
$$

$$
Y=H+\operatorname{FFN}(\operatorname{RMSNorm}(H))
$$

```text
X [T,C]
 |-------------------------------+
 -> RMSNorm -> causal attention -> + = H [T,C]
                                      |---------------------+
                                      -> RMSNorm -> FFN -----+ = Y [T,C]
```

入力と出力の形状が等しいため、ブロックは積み重ねられます。

## 残差接続

入力をサブレイヤーの結果で置き換えるのではなく、次の形にします。

$$
y=x+F(x)
$$

逆伝播は次のようになります。

$$
\frac{\partial L}{\partial x}
=\frac{\partial L}{\partial y}
\left(I+\frac{\partial F}{\partial x}\right)
$$

恒等パスは勾配 (gradient) を深さ方向にそのまま運び、各サブレイヤーが完全な
置き換えではなく補正を学習できるようにします。残差加算の両方のオペランドは
同一の形状でなければなりません。

## pre-norm と post-norm

```text
pre-norm:  x + Sublayer(Norm(x))
post-norm: Norm(x + Sublayer(x))
```

pre-norm は残差の恒等パスを正規化の外側に残すため、深いモデルで勾配の流れが
改善されることがよくあります。一方で表現のスケールが変化するため、ロジットの
前に最終正規化が必要になります。

アーキテクチャの選択は、普遍的なルールではなく、訓練曲線、勾配ノルム、
最終的な品質によって評価されます。

## 位置ごとのフィードフォワードネットワーク

注意機構 (attention) は位置をまたいで情報を混合します。FFN は共有パラメータで
各位置を独立に変換します。

$$
\operatorname{FFN}(x)
=W_2\operatorname{ReLU}(W_1x+b_1)+b_2
$$

```text
[T,C] -> Linear -> [T,F] -> ReLU -> Linear -> [T,C]
```

拡大幅 $F$ が非線形の容量を追加します。注意機構がどこを読むかを選び、FFN は
集めた情報を変換します。後の章では ReLU を現代的なゲート付き活性化関数に
置き換えます。

## パラメータ数

チャネル数 $C$、FFN 幅 $F$ に対して、

- Q/K/V/出力射影: $4(C^2+C)$
- 2 つの RMSNorm スケール: $2C$
- FFN 拡大層: $CF+F$
- FFN 射影層: $FC+C$

合計:

$$
4C^2+2CF+7C+F
$$

幅が大きくなると、行列の項がバイアスや正規化スケールを圧倒します。

## 深さ

ブロックを積み重ねると、関係の上にさらに関係を構築できます。ただし深く
するほど、パラメータ数、FLOPs、活性化メモリ、逐次的なレイテンシ、最適化の
難しさも増します。残差 (residual)、正規化、初期化、学習率スケジュールが、
深いモデルの最適化を実用的なものにします。

## ドロップアウトと正則化

リファレンスのブロックは、決定的な観測を保つためにドロップアウトを省いて
います。ドロップアウトの実装には次が必要です。

- 訓練/評価モード
- 逆伝播のために保存されるシード付きマスク
- 保持された活性化のスケーリング
- 再現可能な RNG 状態

ドロップアウトが有効かどうかはデータとモデルの規模に依存し、測定によって
確かめる必要があります。

## テストされる性質

- 入力と出力の形状が一致する
- パラメータ数が式と一致する
- パラメータ参照が一意である
- 入力とパラメータの勾配が有限である
- ブロック全体がプレフィックスの因果性を保つ
- 不正な幅は構築時に失敗する

ブロック内の別のコンポーネントが時間位置を誤って混合するなら、注意機構の
因果性だけでは不十分です。

## 実装ウォークスルー

`FeedForward` は `expansion: Linear(C,F)` と
`projection: Linear(F,C)` を所有します。最後のチャネル軸を検証し、拡大、
ReLU、射影を適用します。同じ 2 つの層がすべての時間方向の行を処理し、FFN 内
では位置をまたぐ相互作用はありません。

`TransformerBlock.apply` は、隠れた変更なしに 2 つの pre-norm の式をそのまま
写します。

```scala
val afterAttention = input + attention(attentionNorm(input))
afterAttention + feedForward(feedForwardNorm(afterAttention))
```

最初の残差は、正規化後の入力ではなく元の `input` に注意機構の出力を加算
しなければなりません。2 つ目の正規化は `afterAttention` を読まなければ
なりません。注意機構のブランチがすでに残差ストリームを更新しているためです。
こうした区別は長い式の中では見失いやすく、それが中間結果に名前を付けている
理由です。

残差加算の前後では、すべてのテンソルが `[T,C]` を保ちます。形状が変わるのは
注意機構と FFN の内部だけです。`parameters` は順伝播の所有順、すなわち
注意機構の正規化、4 つの注意機構射影、FFN の正規化、拡大層、射影層の順に
従います。安定した順序は、チェックポイントのシリアライズとオプティマイザの
状態のために必要です。

`forwardCached` が異なるのは、注意機構の呼び出しと、入力の時間次元が 1 で
あることを要求する点だけです。RMSNorm と FFN は位置ごとの演算なので、
キャッシュを必要としません。層ごとの KV キャッシュが注意機構に 1 つ渡され
ます。これをブロック間で共有すると、異なるパラメータ集合による射影が混ざって
しまいます。

## テストの読み方

形状保存のテストは、残差や FFN 幅の誤りを捕捉します。パラメータ数は
コンポーネントごとに計算されるため、バイアスや正規化のパラメータが欠けて
いれば失敗します。スカラー出力を入力とすべてのパラメータへ逆伝播させ、
グラフの接続性を検証します。ブロック全体の因果性テストは、注意機構の外側での
未来の混合を捕捉します。不正な隠れ幅は構築時に失敗します。

## デバッグチェックリスト

1. 各ブランチの前後で残差ストリームに名前を付ける。
2. 正規化の入力と残差の加算対象が誤って同じテンソルになっていないことを
   検証する。
3. 加算の前にブランチ出力が `[T,C]` であることを確認する。
4. あるパラメータの勾配がゼロの場合は、そのブランチの出力がスカラー損失
   (loss) に到達しているかを追跡する。
5. 因果性が破れている場合は、注意機構と FFN の行独立性を別々にテストする。

## 演習

1. `C=768`、`F=3072` のパラメータ数を数える。
2. 残差を取り除き、深いモデルでの入力勾配ノルムを比較する。
3. post-norm を実装し、同一シードでの訓練を比較する。
4. ReLU を tanh に置き換え、活性化と勾配の統計を測定する。
5. FFN の行が独立であることをテストする。
6. ドロップアウトのモードと RNG のプロトコルを設計する。

## 完了条件

- 2 つの pre-norm 残差の式を書ける。
- 残差の恒等パスを説明できる。
- 注意機構と FFN の役割を区別できる。
- 拡大層と射影層の形状を追跡できる。
- ブロックのパラメータ数を計算できる。
- ブロック全体の因果性を再テストする理由を説明できる。
- `TransformerBlockSuite` が通る。
