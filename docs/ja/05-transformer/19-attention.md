# 19 — スケール付き内積の causal self-attention

## この章で作るもの

クエリ・キー・バリューの射影、スケール済みスコア、因果マスク (causal mask)、
行ごとの softmax、マルチヘッドの分割と連結、そして出力射影を実装します。
ソース: `src/main/scala/learnai/transformer/Attention.scala`。

テストでは、行が正規化されていること、未来位置の重みがゼロであること、
プレフィックス不変性、そしてプレフィックスのみの損失 (loss) から未来の入力への
勾配 (gradient) がゼロであることを検証します。

## attention が解決すること

初期のトークン表現には、そのトークンと位置の情報しか含まれません。文脈を
扱うには、内容に基づいて過去の位置を選択する必要があります。self-attention
(self-attention) では、各隠れベクトル $x_t$ から次を作ります。

$$
q_t=x_tW_Q,\qquad k_t=x_tW_K,\qquad v_t=x_tW_V
$$

- クエリ: この位置が探しているもの
- キー: ある位置がマッチしうるもの
- バリュー: 選択後に転送される情報

クエリとキーはどこを読むかを選び、バリューは何が読まれるかを決めます。

## 形状を追う

1 つの系列について、時間長 $T$、チャネル数 $C$ とすると、

```text
X: [T,C]
Wq, Wk, Wv: [C,C]
Q, K, V: [T,C]
scores = Q K^T: [T,T]
weights = softmax(mask(scores)): [T,T]
output = weights V: [T,C]
```

スコアの行は読み手であるクエリ位置に、列は候補となるキー位置に対応します。

## 内積による類似度

$$
s_{ij}=q_i\cdot k_j
$$

訓練 (training) によって、有用な関係のためにクエリとキーの方向を揃えることが
できます。内積は大きさの影響を含むため、コサイン類似度と同一ではありません。

## なぜ $\sqrt d$ で割るのか

ヘッド幅 $d$ に対して、独立で分散 1 のチャネル同士の積を考えると、内積の分散は
$d$ に比例して増大します。大きなスコアは softmax を飽和させ、有用な勾配を
縮小させます。そこでスコアをスケールします。

$$
s_{ij}=\frac{q_i\cdot k_j}{\sqrt d}
$$

これにより、初期のスコアのスケールがヘッド幅によらず安定しやすくなります。

## 因果マスク

位置 $i$ は未来の位置 $j>i$ を読んではなりません。

```text
query 0: [0,-,-]
query 1: [0,1,-]
query 2: [0,1,2]
```

$$
\tilde s_{ij}=\begin{cases}
s_{ij}&j\le i\\
-\infty&j>i
\end{cases}
$$

実装では、有限値の不変条件を保つために有限の `-1e9` を使います。安定化された
softmax の後では、確率ゼロへとアンダーフローします。逆伝播でも上三角部分は
ブロックされます。

マスクがないと、訓練時に次の入力位置にすでに存在するターゲットトークンを
読めてしまい、直接的なリークが生じます。

## 行ごとの softmax とバリューの集約

$$
a_{ij}=\frac{e^{\tilde s_{ij}}}{\sum_r e^{\tilde s_{ir}}}
$$

各行は、許可されたキーの上の確率分布です。出力は重み付き和です。

$$
o_i=\sum_j a_{ij}v_j
$$

softmax の逆伝播ではヤコビ行列を構築しません。出力確率 $y_i$ と上流勾配
$g_i$ に対して、

$$
\frac{\partial L}{\partial x_i}
=y_i\left(g_i-\sum_jg_jy_j\right)
$$

すべてのロジットに定数を加えても softmax の確率は変わらないため、行の勾配の
合計はゼロになります。

## multi-head attention

チャネルを幅 $d=C/H$ の $H$ 個のヘッドに分割します。

```text
project Q/K/V: [T,C]
split:         H x [T,d]
attention:     H x [T,d]
concatenate:   [T,C]
project:       [T,C]
```

ヘッドごとに異なる関係を学習できます。アーキテクチャが人間に読める役割を
あらかじめ割り当てるわけではありません。`C` は `H` で割り切れなければ
なりません。

この教育用エンジンでは列を明示的にスライスします。本番のカーネルは
`[batch,head,time,d]` に reshape し、バッチ化された行列積を使います。

## 3 つの因果性テスト

1. **重み:** 狭義の上三角部分がゼロである。
2. **順伝播:** 未来の入力を変更しても、それより前の出力は変化しない。
3. **逆伝播:** プレフィックスのみの損失では、未来の入力への勾配がゼロになる。

それぞれが異なる種類のマスク不具合を捕捉します。

## 計算量

attention は、ヘッド・バッチあたり $O(T^2C)$ の計算時間と
$O(T^2)$ の重み行列メモリを要します。コンテキストを 2 倍にすると、スコアの
要素数は 4 倍になります。

Flash Attention は、タイル化とオンラインでの softmax 計算により、厳密な
attention 計算を保ったまま、スコア行列全体を高帯域メモリに実体化せずに済むように
します。

## 実装ウォークスルー

`CausalSelfAttention.forwardWithWeights` は、形状と時間長が空でないことの
検証から始まります。位置間で共有される 4 つの全結合層、すなわちクエリ、キー、
バリュー、最終出力の射影を適用します。Q、K、V はいずれも形状 `[T,C]` です。

各ヘッドでは、`from` と `until` が幅 `D = C/H` の連続したチャネルスライスを
1 つ選択します。ヘッドの計算は次のとおりです。

```scala
val scores = headQuery.matmul(headKey.transpose2D).scale(scale)
val weights = scores.causalMask().softmaxRows
val headOutput = weights.matmul(headValue)
```

コードを暗記するのではなく、形状を追ってください。

```text
Qh [T,D] x Kh^T [D,T] -> scores [T,T]
weights [T,T] x Vh [T,D] -> head output [T,D]
H head outputs concatenated -> [T,C]
output projection -> [T,C]
```

因果マスクは、softmax の前に、キー位置が `j > i` となる要素を置き換えます。
その逆伝播は、許可されたセルにだけ勾配を通します。各クエリ位置がキー位置の
上の分布を 1 つ必要とするため、softmax は行ごとに適用されます。

`Tensor.concatenateColumns` は元のチャネル順序を復元します。代わりに行を
連結すると、要素数はもっともらしくても意味は誤ったものになります。出力射影は、
連結後にヘッド間で情報を混合します。

キャッシュ版のメソッドは、1 つのクエリ行に対して同じ式を実装します。まず
現在のキーとバリューを追加し、キャッシュ内の各位置に対するヘッドごとの
スコアを計算し、最大値を引いて正規化し、重み付けされたヘッドチャネルを
書き込み、同じ出力射影を適用します。生成は順伝播のみであるため、キャッシュ
された配列は意図的に自動微分から切り離されています。

## テストの読み方

行の合計と未来位置が厳密にゼロであることの確認により、マスクと softmax の
順序を検証します。未来の入力を変更しながらそれより前の出力を比較するテストは、
重みを調べることなく因果性を検証します。プレフィックスのみの損失による勾配
テストはさらに強力で、未来の入力への勾配は厳密にゼロでなければなりません。
ヘッド分割のテストは 3 で割り切れる幅を使います。キャッシュ付き attention は、
独立した完全パスとすべてのプレフィックスで比較されます。不正なヘッド分割と
キャッシュあふれのテストが、構築時と実行時の境界をカバーします。

## デバッグチェックリスト

1. すべての中間結果の横に `[T,C]`、`[T,D]`、`[T,T]` を書き込む。
2. ヘッドを結合する前に、attention の 1 行を調べる。
3. マスクが softmax より前に行われ、クエリ位置より大きいキー位置を対象に
   していることを検証する。
4. すべての行の合計と未来のセルを確認する。
5. キャッシュ版と完全版の結果が異なる場合は、射影された Q、追加された K/V、
   スコアの順序、位置の長さ、出力射影の順に比較する。

## 演習

1. `T=4` の因果マスクを書き下す。
2. 自分で選んだ Q/K/V から 1 つのヘッドを手計算する。
3. スケーリングを取り除き、ヘッド幅が大きくなるにつれて softmax のエントロピー
   がどう変わるかを測定する。
4. softmax の後にマスクすると行の正規化が壊れる理由を示す。
5. softmax の逆伝播をヤコビ行列から導出する。
6. ヘッド数を変えても Q/K/V のパラメータ数が変わるとは限らない理由を説明する。
7. attention の重みがモデルの完全な説明にはならない理由を説明する。

## 完了基準

- Q、K、V の役割と形状を説明できる。
- `[T,T]` スコアの両方の軸を解釈できる。
- $1/\sqrt d$ による分散のスケーリングを説明できる。
- マスクが softmax より前に行われる理由を説明できる。
- マルチヘッドの分割と連結を追跡できる。
- attention の因果性と勾配に関するすべてのテストが通る。
