# 17 — バイグラム言語モデル

## この章で作るもの

現在のトークンから次のトークンを予測する訓練可能なモデルを、Tensor の
ルックアップ、融合されたクロスエントロピー、AdamW による訓練、自己回帰
サンプリングとともに実装します。

- モデル: `src/main/scala/learnai/lm/BigramLanguageModel.scala`
- Tensor 演算: `gatherRows` と `crossEntropy`

この章では、トークナイザ (tokenizer)、訓練ペア、ロジット、損失 (loss)、
逆伝播、オプティマイザ、生成を 1 つの言語モデルパイプラインへと結合
します。

## バイグラム仮定

自己回帰モデルは系列確率を次のように分解します。

$$
p(x_1,\ldots,x_T)=\prod_{t=1}^{T}p(x_t\mid x_{<t})
$$

バイグラムは、過去全体を現在のトークンだけに単純化します。

$$
p(x_{t+1}\mid x_0,\ldots,x_t)
\approx p(x_{t+1}\mid x_t)
$$

長いコンテキストはモデル化できませんが、次トークン予測という目的関数の
全体像を余すところなく見せてくれます。

## 遷移ロジットテーブル

語彙サイズ $V$ に対して、訓練可能な
$W\in\mathbb{R}^{V\times V}$ を保持します。行 $i$ には、現在のトークンが
$i$ のときの次トークンのロジットが入ります。

```text
W shape: [current token, next token] = [V,V]
```

入力 ID `[N]` は行を集めてロジット `[N,V]` を作ります。繰り返し現れる ID
は 1 つの共有された行を読み、逆伝播時に対応するすべての勾配 (gradient) を
蓄積します。埋め込み (embedding) のルックアップも、これと同じ
gather/scatter の仕組みを使います。

このテーブルは $V^2$ 個のパラメータを含みます。トークンが 50,000 個なら
25 億個になり、言語モデルがトークンをより低次元の隠れ空間を通して分解
する理由を示しています。

## 融合されたクロスエントロピー

ロジット $z$ とターゲットクラス $t$ に対して:

$$
L=\log\sum_j e^{z_j}-z_t
$$

その勾配は:

$$
\frac{\partial L}{\partial z_j}=p_j-\mathbb{1}[j=t]
$$

`Tensor.crossEntropy` は、安定な log-sum-exp とこの導関数を、one-hot の
Tensor や独立した softmax グラフを実体化せずに組み合わせます。これは
演算融合 (operation fusion) の小さな一例です。

すべての勾配の行は合計がゼロになります。

$$
\sum_j(p_j-\mathbb{1}[j=t])=1-1=0
$$

テストはこの性質、大きなロジットに対する安定性、有限差分を確認します。

## 訓練ペア

```text
tokens:  [t0,t1,t2,t3]
inputs:  [t0,t1,t2]
targets: [t1,t2,t3]
```

フルバッチの 1 ステップは次のとおりです。

```scala
val loss = model.loss(inputs, targets)
loss.backward()
optimizer.step(model.parameters)
```

繰り返し現れるターゲット遷移は、他の候補に対して相対的にそのロジットを
引き上げます。

## 自己回帰生成

訓練では真の直前トークンを使います。これは教師強制 (teacher forcing) と
呼ばれます。生成では、モデル自身のサンプルを次の入力として戻します。

```text
start -> logits row -> softmax -> sample -> use sample as next input -> repeat
```

低確率のサンプルは、ほとんど訓練されていない経路に入り込み、誤りを
積み重ねる可能性があります。したがって訓練と生成は、異なる入力分布を
見ることになります。

## 温度

$$
p_i=\operatorname{softmax}(z_i/\tau),\quad\tau>0
$$

- 1 より小さい: 高いロジットに集中する。
- 1 に等しい: 学習した分布のまま。
- 1 より大きい: より平坦で多様になる。
- ゼロに近づく: argmax に近づく。

## UTF-8 に関する注意

バイト/BPE トークンの任意の組み合わせが有効な UTF-8 になるとは限りません。
コンテキスト長 1 のモデルでは特にそうです。このデモは厳格なデコードを
使い、置換文字で隠すのではなく失敗を報告します。

## バイグラムから Transformer へ

バイグラムは、最後のトークンが同じであれば、それより前のコンテキストが
変わっても同一の予測を返します。Transformer は注意機構 (attention) で
前方の位置を混ぜ合わせ、文脈に応じた隠れベクトルを作ります。次の要素は
変わらずそのまま残ります。

- トークン化と因果的シフト。
- ロジットとクロスエントロピー。
- 逆伝播とオプティマイザ。
- 自己回帰生成。

## 実装ウォークスルー

`BigramLanguageModel` は、形状 `[V,V]` の訓練可能な行列を 1 つ所有します。
行 `i` には、入力トークン `i` の後に来る次トークンのロジットが入ります。
したがってモデルの適用は、行列積ではなく埋め込み方式の行 gather です。

語彙 `{A,B,C}` について、望むサイクルが `A->B`、`B->C`、`C->A` だとします。
訓練ペアは対応する 3 つの行/ターゲットの選択です。クロスエントロピーは、
選択された各行でターゲット列のロジットを増やし、競合を減らします。隠れ
状態が存在しないため、現在のトークンが同じ 2 つの位置は、それ以前の履歴に
かかわらず常に同じ次トークン分布を生みます。

`loss` は入力とターゲットの長さと範囲を検証し、遷移行を集めて Tensor の
クロスエントロピーを呼びます。`gatherRows` は繰り返された選択を同じ行列
の行へ振り向けるため、入力トークンが複数回現れる場合、逆伝播は勾配を
蓄積します。

生成は検証済みの開始トークンから始まります。各ステップで、最後に生成
された ID を選び、その行のロジットを安定な softmax 分布に変換し、呼び出し
側の RNG でサンプリングして追加します。返されるベクトルには開始トークン
が含まれます。新規トークンをゼロ個要求した場合は、検証済みのその 1 個の
トークンだけがちょうど返されます。

`BigramTrainer` は毎ステップ損失グラフを構築し直し、AdamW を通じて唯一の
パラメータを更新します。この小さなモデルは、トークン ID から損失、逆伝播、
オプティマイザ、シード付き生成までを貫く最初の完全な経路です。

## テストの読み方

ロジットのルックアップは、出力の形状と行を正確に確認します。3 トークンの
サイクルには既知の学習可能な最適解があり、大幅な損失の減少と支配的な
ターゲット確率を検証します。同じ生成シードのテストは RNG の所有権を
確かめます。長さゼロの生成はループ境界を捕捉します。不正な ID と温度は、
インデックス参照や softmax の前に失敗します。

## デバッグチェックリスト

1. 失敗した予測は、1 つの行と 1 つのターゲット列として解釈する。
2. ターゲットがちょうど 1 位置ずれていることを検証する。
3. 繰り返された入力 ID が 1 つの行の勾配に蓄積されることを確認する。
4. 損失は下がるのに生成が繰り返しばかりに見える場合は、バイグラムの
   コンテキストがちょうど 1 トークンであることを思い出す。
5. ロジット、確率、サンプリングされた ID を別々の段階として比較する。

## 演習

1. 語彙サイズ 3 のパラメータ数を数える。
2. 経験的な遷移カウントと学習された softmax 確率を比較する。
3. ターゲットのロジットを 1 増やし、損失の変化を測定する。
4. いくつかの温度でトークン頻度を比較する。
5. 別に用意した検証ペアの損失を計算する。
6. バイグラムが区別できない 2 つのコンテキストを見つける。
7. 遷移テーブルを低ランク積で置き換える。
