# 10 — リバースモード自動微分

## この章で作るもの

1 回の逆向き走査ですべてのパラメータの勾配 (gradient) を計算する、
スカラーの計算グラフを作ります。ソース:
`src/main/scala/learnai/autodiff/Value.scala`。

## 式をグラフとして扱う

次の式を考えます。

$$
x=2,\quad y=-3,\quad z=xy+x^2
$$

通常の評価では結果しか残らないかもしれません。自動微分 (automatic
differentiation) は、演算をノードとして、依存関係をエッジとして記録
します。

```text
x ----(*)----+
 \   /       |
  \ y        (+)---- z
   \         |
    --pow(2)-+
```

各 `Value` ノードは次を保持します。

- 順伝播の `data`。
- 最終出力のこのノードに関する勾配。
- 演算と親ノード。
- 局所的な逆伝播規則。

## フォワードモードとリバースモード

- フォワードモード (forward mode) は、1 つの入力からすべての出力へ
  向けて微分を伝播する。
- リバースモード (reverse mode) は、1 つの出力からすべての入力へ向けて
  伝播する。

ニューラルネットワークには多数のパラメータと 1 つのスカラー損失 (loss)
があります。リバースモードは、1 回の逆向き走査ですべてのパラメータ勾配
を得ます。

## 局所微分と上流勾配の積

$c=f(a,b)$ について、$\partial L/\partial c$ が上流から届くとき:

$$
\frac{\partial L}{\partial a}
=\frac{\partial L}{\partial c}\frac{\partial c}{\partial a}
$$

乗算 $c=ab$ の局所微分は $b$ と $a$ です。

```scala
output.backwardRule = () =>
  accumulateGradient(other.data * output.gradient)
  other.accumulateGradient(data * output.gradient)
```

`output.gradient` は上流からの情報で、オペランドの値が局所微分です。

## 上書きではなく累積する

$z=x^2+x$ では、`x` は 2 つの経路を通って出力に到達します。

$$
\frac{dz}{dx}=2x+1
$$

一方の経路は `2x` を、もう一方は `1` を寄与します。上書きすると経路が
失われるため、勾配は累積しなければなりません。パラメータ共有、残差
接続、埋め込み (embedding) の繰り返しルックアップは、すべてこの規則に
依存しています。

## トポロジカル順序

あるノードの逆伝播規則は、下流のすべての経路が寄与を終えた後に実行
されなければなりません。トポロジカル順序を構築し、それを逆順に走査
します。

```text
forward order:  leaves -> intermediates -> loss
backward order: loss   -> intermediates -> leaves
```

共有されるノードはオブジェクトの同一性で追跡します。根の勾配は次の値で
初期化します。

$$
\frac{\partial L}{\partial L}=1
$$

## 局所規則

| 順伝播 | 局所微分 |
| --- | --- |
| $a+b$ | $1,1$ |
| $ab$ | $b,a$ |
| $a^r$ | $ra^{r-1}$ |
| $e^a$ | $e^a$ |
| $\log a$ | $1/a$ |
| $\tanh a$ | $1-\tanh^2a$ |
| $\operatorname{ReLU}(a)$ | `a>0` なら `1`、それ以外は `0` |

減算と除算は既存の演算から合成します。

$$
a-b=a+(-1)b,\qquad a/b=ab^{-1}
$$

## パラメータ更新とグラフの寿命

演算の data は順伝播時点のスナップショットです。正しい訓練ステップは
次のとおりです。

1. 現在のパラメータから新しいグラフを構築する。
2. スカラー損失に対して `backward()` を呼ぶ。
3. パラメータの葉を更新する。
4. 古いグラフを破棄する。
5. 次のステップのために順伝播を作り直す。

葉を更新しても、古い中間値が遡って再計算されることはありません。

## 勾配チェック

同じ関数を生の `Double` と `Value` で実装し、自動微分の勾配と中心差分の
勾配を比較します。

```scala
val numeric = Calculus.derivative(rawFunction, at = x.data)
output.backward()
Assert.close(x.gradient, numeric, tolerance = 1e-8)
```

手計算による導出、有限差分、自動微分は、互いに独立した 3 つの視点です。

## 単純化している点

- スカラーノード 1 つにつき 1 つの JVM オブジェクト。
- 再帰によるグラフの順序付け。
- 高階微分なし。
- インプレースや並列の逆伝播なし。
- グラフのプロファイラや可視化なし。

Tensor エンジンは、多数のスカラーを 1 つの演算ノードにまとめます。

## 実装ウォークスルー

`Value` は 2 種類のノードを区別します。`Value.parameter` は、最適化の
途中で data が変わりうる訓練可能な葉を作ります。`Value.constant` は
訓練不可能な葉を作ります。すべての算術メソッドは、順伝播の data を即座
に計算し、親への参照を持つ演算ノードを作り、上流勾配をそれらの親へ送る
方法を知っているクロージャを設定します。

`x=2`、`y=-3` で `z = x * y + x.pow(2)` を追跡します。

```text
m = x*y   = -6
p = x^2   = 4
z = m+p   = -2
```

逆伝播は、深さ優先走査でノードを発見することから始まります。
`IdentityHashMap` を使うのは、2 つの異なるノードが等しい data とラベル
を持ちながらも、別々のグラフ頂点であり続けなければならないからです。
トポロジカル順序には各オブジェクトが 1 回だけ現れます。勾配をクリアし、
`z.gradient` を 1 に初期化して、クロージャを逆順に実行します。

加算のクロージャは `1 * upstream` を `m` と `p` の両方へ送ります。乗算
は `y * upstream` を `x` へ、`x * upstream` を `y` へ送ります。べき乗は
`2*x * upstream` を `x` へ送ります。`x` への 2 つの寄与は累積されます。

```text
from x*y: y = -3
from x^2: 2*x = 4
total dz/dx = 1
dz/dy = x = 2
```

`applyGradient` は訓練可能な葉に限定されています。演算ノードを更新
すると、保持している data と親との関係が壊れてしまいます。グラフの
data は順伝播時点のスナップショットなので、訓練では更新のたびに新しい
グラフを構築します。

## テストの読み方

積と和のテストは、局所規則を手計算で確認します。ノード再利用のテスト
は、累積の代わりに代入してしまうバグの最小の検出器です。合成関数は、
グラフとは独立な実装である中心差分で検証します。2 回目の `backward()`
呼び出しは、古い勾配がクリアされることを検証します。`log` の定義域
テストは、逆伝播の前に不正な順伝播値を捕まえます。

## デバッグチェックリスト

1. グラフを描き、共有されるオブジェクトの同一性に印を付ける。
2. トポロジカル順序を出力し、各ノードが 1 回だけ現れることを確認する。
3. スカラーの根だけに勾配 1 を設定する。
4. 共有ノードの勾配が小さすぎる場合は、累積の代わりの代入を探す。
5. 有限差分が一致しない場合は、局所規則を 1 つずつ比較する。
6. パラメータ更新の後はグラフを作り直す。

## 演習

1. $(x+2)^3$ を手計算と `Value` の両方で微分する。
2. `exp` と算術演算からシグモイドを合成する。
3. `sin` を追加し、数値的に検証する。
4. 勾配の累積を代入に置き換え、共有ノードでの失敗を観察する。
5. DOT グラフのエクスポータを設計する。

## 完了基準

- 式のノードとエッジをコードに対応付けられる。
- リバースモードが多入力・1 出力に適する理由を説明できる。
- 上流勾配と局所微分を区別できる。
- 勾配の累積が必要になる式を挙げられる。
- 更新後にグラフを作り直す理由を説明できる。
- `ValueSuite` が通る。
