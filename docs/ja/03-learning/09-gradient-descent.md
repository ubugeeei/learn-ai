# 09 — 勾配降下法で学習を組み立てる

## この章で作るもの

1 パラメータの関数を最小化し、各ステップでパラメータ、損失 (loss)、
勾配 (gradient) を記録します。ソース:
`src/main/scala/learnai/learning/GradientDescent.scala`。

## 学習を最適化として書く

パラメータ $\theta$ がモデルの挙動を制御し、損失 $L(\theta)$ が誤差を
測るとします。訓練 (training) は、損失を最小化するパラメータを探索する
ことです。

$$
\theta^*=\operatorname*{arg\,min}_{\theta}L(\theta)
$$

`min` は最小値そのものです。`arg min` はそれを与える引数です。

最初の例では、パラメータを `3` に向けて動かします。

$$
L(\theta)=(\theta-3)^2
$$

この損失は非負で、$\theta=3$ のときに限りゼロになります。導関数は
次のとおりです。

$$
\frac{dL}{d\theta}=2(\theta-3)
$$

## 更新則

勾配は損失が増加する方向を指すため、それを引き算します。

$$
\theta_{t+1}=\theta_t-\eta\frac{dL}{d\theta_t}
$$

| 記号 | 意味 | コード |
| --- | --- | --- |
| $t$ | ステップ | `step` |
| $\theta_t$ | 現在のパラメータ | `parameter` |
| $L$ | 損失関数 | `loss` |
| $dL/d\theta_t$ | 局所勾配 | `gradient` |
| $\eta$ | 学習率 (learning rate) | `learningRate` |

```scala
parameter -= learningRate * gradient
```

この順序が重要です。

1. 現在のパラメータで順伝播を実行する。
2. 損失を計算する。
3. 勾配を求める。
4. パラメータを更新する。
5. 新しいパラメータで次のステップを始める。

## 軌跡を観察する

```console
$ nix develop -c sbt 'runMain learnai.learning.runGradientDescentLab'
```

最終値だけでなく、次の点を調べます。

- 損失は一貫して下がっているか。
- 勾配の大きさはゼロに近づいているか。
- パラメータが 2 点の間で振動していないか。
- どこかの値が `NaN` や無限大になっていないか。
- 改善が小さくなりすぎていないか。

`DescentObservation` は、これらの問いに答えるために必要な証拠を保存
します。

## 学習率のトレードオフ

- 小さすぎる: 安定するが遅い。
- 適切: 速く安定した減少。
- 大きすぎる: 行き過ぎ、振動、あるいは発散。

この 2 次関数の例では、目標までの距離は各ステップで $1-2\eta$ 倍に
なります。`η=0.1` では `0.8` 倍に縮み、`η=1` では縮まずに符号だけが
反転し、`1` を超えると増大します。

## 非凸な損失

この例は凸で、最小値は 1 つです。ニューラルネットワークの損失曲面は
高次元で、鞍点、平坦な領域、多数の経路を含みます。勾配降下法 (gradient
descent) は局所的な傾きしか見ないため、大域的最小値は保証されません。

実際の訓練は、初期化、ミニバッチのノイズ、オプティマイザ (optimizer) の
状態、巨大なパラメータ空間の組み合わせです。成功は理論的な最小値への
到達だけでなく、未知のデータでの評価によって判断します。

## 訓練・検証・テストデータ

- 訓練データはパラメータを更新する。
- 検証データはハイパーパラメータの選択と停止の判断に使う。
- テストデータは最終評価のために取っておく。

テスト結果への適応を繰り返すと、テストの情報が開発に漏れ、汎化性能を
過大に見積もってしまいます。

## 実装ウォークスルー

`GradientDescent.minimize` は、初期スカラー値、損失関数、その導関数、
学習率、ステップ数を受け取ります。導関数はオプティマイザの内部に隠す
のではなく注入する設計なので、更新則は目的関数から独立に保たれます。

各反復では、同じ現在のパラメータで損失と勾配を評価し、
`DescentObservation` を記録してから初めて更新します。

```text
observe (step, parameter, loss, gradient)
parameter = parameter - learningRate * gradient
```

更新の前に記録することが重要です。もし損失を更新後に、勾配を更新前に
記録すると、1 つの行が 2 つの異なる点を記述することになり、学習曲線は
誤解を招くものになります。

(L(w)=(w-3)^2)、(w_0=0)、学習率 `0.25` の場合:

```text
gradient at 0 = 2(0-3) = -6
w1 = 0 - 0.25*(-6) = 1.5
loss falls from 9 to 2.25
```

この点での負の勾配は上り方向を指しているため、それを引くと `3` に
向かって進みます。学習率 `1` では列が `0` と `6` の間で振動し、`1` より
大きい学習率では大きさが増大します。学習率が見た目だけのオプションでは
なく、アルゴリズムの一部であるのはこのためです。

コードは、ステップ数、学習率、初期値、そして計算されるすべての損失・
勾配・更新値を検証します。非有限の値が現れた場合は、残りの軌跡を汚染
する代わりに、最初の不正な反復で停止します。

## テストの読み方

凸な 2 次関数は、手計算で分かる最適値と、選んだ学習率での損失の単調
減少という期待を与えてくれます。ゼロステップのテストは初期観測だけが
返ることを確認し、ループの off-by-one エラーを捕まえます。不正な
ハイパーパラメータのテストは、負のステップ数や学習率がユーザー関数を
呼び出す前に失敗することを保証します。

## デバッグチェックリスト

1. 同じ観測からパラメータ、損失、勾配を出力する。
2. 勾配を引くことで、手計算で分かる最適値へ向かうかを確認する。
3. ニューラルネットワークをデバッグする前に、問題を 2 次関数へ縮小する。
4. 損失が振動する場合は、学習率と曲率を比較する。決定的な目的関数で
   ランダム性のせいにしない。
5. 最初の非有限の更新を特定し、その勾配の大きさを調べる。

## 演習

1. 学習率 `0.01`、`0.5`、`1.0`、`1.1` を比較する。
2. `3` の両側から開始し、勾配の符号を説明する。
3. $(\theta+2)^2+1$ を最小化し、その最適値を予測する。
4. 履歴を CSV として出力し、プロットする。
5. 解析的な導関数を数値微分に置き換える。

- 更新式のマイナス符号を説明できる。
- 小さすぎる学習率と大きすぎる学習率を実際に示せる。
