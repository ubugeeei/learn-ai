# 06 — 行列と形状

## この章で作るもの

転置 (transpose)・行列ベクトル積・行列積を備えた、行優先 (row-major) で不変な
`MatrixD` を作ります。ソース: `src/main/scala/learnai/math/MatrixD.scala`。

## 行列は二つの軸を持つ

$m$ 行 $n$ 列の行列は次のとおりです。

$$
\boldsymbol{A}=
\begin{bmatrix}
a_{11}&\cdots&a_{1n}\\
\vdots&\ddots&\vdots\\
a_{m1}&\cdots&a_{mn}
\end{bmatrix}
\in\mathbb{R}^{m\times n}
$$

その形状 (shape) は `[m,n]` です。Scala では、これらの次元は `matrix.rows` と
`matrix.columns` です。

```scala
val matrix = MatrixD.fromRows(
  Vector(
    VectorD(1.0, 2.0, 3.0),
    VectorD(4.0, 5.0, 6.0)
  )
)
```

形状は `[2,3]` で、`matrix(1,2)` は `6.0` を返します。LLM は埋め込み
(embedding) テーブルを `[vocabulary,channels]` として、密な重みを
`[inputChannels,outputChannels]` またはその文書化された転置として保持します。

## 行優先レイアウト

```text
matrix:  [[a,b,c],
          [d,e,f]]
storage: [a,b,c,d,e,f]
offset(row,column) = row * columns + column
```

一次元の連続したストレージはオブジェクト数が少なく、連続する行へのアクセスが
キャッシュに優しくなります。`MatrixD` は次元情報を保持しているため、フラットな
ストレージが二次元の意味を失うことはありません。

## 転置

転置は行と列を入れ替えます。

$$
(\boldsymbol{A}^{\mathsf T})_{ij}=A_{ji}
$$

形状は `[m,n] -> [n,m]` と変わり、二度転置すると元に戻ります。

$$
(\boldsymbol{A}^{\mathsf T})^{\mathsf T}=\boldsymbol{A}
$$

このような常に成り立つ関係は、プロパティテストとして有用です。

## 行列ベクトル積

`A [m,n]` と `x [n]` に対して:

$$
\boldsymbol{y}=\boldsymbol{A}\boldsymbol{x},\qquad
y_i=\sum_{j=1}^{n}A_{ij}x_j
$$

```text
[m,n] x [n] -> [m]
```

出力の各要素は、行列の一つの行と入力ベクトルの内積です。内側の次元は一致しな
ければなりません。ニューラルネットワークでは、行列の一つの行を一つの出力
ニューロンの重みとして解釈できます。

## 行列積

`A [m,k]` と `B [k,n]` に対して:

$$
\boldsymbol{C}=\boldsymbol{A}\boldsymbol{B},\qquad
C_{ij}=\sum_{r=1}^{k}A_{ir}B_{rj}
$$

```text
[m,k] x [k,n] -> [m,n]
       ^   ^
       inner dimensions match
```

出力の各要素は、左の行列の行と右の行列の列の内積です。素朴なアルゴリズムの
計算時間は $O(mkn)$、出力メモリは $O(mn)$ です。最適化されたライブラリは
ブロッキング、ベクトル命令、スレッド、GPU を使いますが、計算する式は同じです。

## 実行時の形状

`MatrixD` は `[2,3]` でも `[4,8]` でも Scala 上の型は同じです。したがって実行時の
次元には実行時チェックが必要になります。エラーメッセージには両方の形状を含めて
います。

型に形状を埋め込める場合もありますが、バッチ長やシーケンス長は実行時にしか
分からないことが多いのです。静的な検証と動的な検証は相補的です。

## 空の次元

`[0,3]` と `[2,0]` はどちらも要素数がゼロですが、意味は異なります。次元を明示
することでその区別が保たれます。最大値や平均のような一部のリダクションは未定義の
ままであり、エラー値を返さなければなりません。

## 実装ウォークスルー

`MatrixD` は `rows`、`columns`、そして一つの行優先配列を保持します。フラットな
オフセットは

```scala
row * columns + column
```

であり、同じ行の隣り合う列は連続しています。すべてのアクセサは、オフセットを
計算する前に行と列を検証します。構築時には `values.length == rows * columns` を
検証するため、それ以降に形状とストレージが食い違うことはありません。

次の行列で行列積を追ってみましょう。

```text
A = [[1, 2],       B = [[5, 6],
     [3, 4]]            [7, 8]]
```

結果の要素 `C(1,0)` は、`A` の行 1 と `B` の列 0 の内積です。

```text
3*5 + 4*7 = 43
```

実装のループの入れ子は、出力の行、出力の列、内側のインデックスの順です。内側の
ループが共有次元を動かし、まさにその内積を構成する積を累積します。出力ストレージ
の要素数は `rows * other.columns` です。互換性は確保の前に `columns == other.rows`
で一度だけ検査されます。

転置は新しい配列を確保し、`(row,column)` を `(column,row)` に対応づけます。
ストライドを使ったビューにすればコピーを避けられますが、この最初の値型と以降の
すべての演算を複雑にしてしまいます。`updated` はベクトルと同じパターンです。
クローンして、変更して、返します。ゼロサイズの次元は、フラットなストレージが
空でも保持されます。`[0,3]` と `[0,5]` は異なる互換性情報を持つからです。

## テストの読み方

インデックスのテストはすべての要素を列挙するため、列優先のバグが対称行列の陰に
隠れることはできません。乗算のテストは、手計算で答えを導いた小さな整数行列を
使います。非互換な形状のテストは、インデックス例外を待つのではなく公開エラーを
検査します。二重転置は有用なプロパティテストです。ゼロ次元のテストは、値だけ
から形状を推論する実装を捕捉します。

## デバッグチェックリスト

1. 値を読む前に、両オペランドの形状を `[m,k] x [k,n]` の形で書き出す。
2. 誤った出力セルを一つ選び、その内積を正確に展開して確認する。
3. 最初の 2 行のフラットオフセットを出力し、行と列の取り違えを見つける。
4. 転置が正方行列でしか動かないなら、すぐに `[2,3]` でテストする。
5. 空の行列が形状を失うなら、ネストしたコレクションから次元を推論する
   コンストラクタを調べる。

## 演習

1. `[2,3] x [3,4]` の出力形状と、要素あたりの乗算回数を答えてください。
2. 行列のアダマール積を追加してください。
3. 単位行列のファクトリを実装し、`A.matmul(I) == A` をテストしてください。
4. 列ごとの合計を返す操作を、まず出力形状を述べてから実装してください。
5. 素朴なループが右の行列を非連続に読む理由を説明してください。

## 完了条件

- 行列の座標を行優先のオフセットに対応づけられる。
- 行列ベクトル積と行列積の式をループのインデックスに対応づけられる。
- 行列の内側の次元が一致しなければならない理由を説明できる。
- `[m,k] x [k,n] -> [m,n]` を資料を見ずに言える。
- `MatrixDSuite` が通る。
